# easy_console — シリアルコンソール仕様書

## 概要

macOS 向けの Rust 製シリアルコンソール TUI アプリケーション。
USB-シリアル変換アダプタや内蔵 UART を通じてデバイスと双方向通信を行う。
組み込み開発・デバッグ用途を主ターゲットとし、開発者が日常的に使いやすい機能を重視する。

---

## 目標

- macOS で `/dev/tty.*` / `/dev/cu.*` デバイスに接続できる
- ボーレート・パリティ・ストップビット等を柔軟に設定できる
- キーボードから文字を送信し、受信データをリアルタイムで表示する
- ログをファイルに保存できる
- 設定をプロファイルとして保存・切り替えできる
- 開発者向けの診断・デバッグ機能を充実させる

## 機能一覧

### 1. シリアルポート管理

- 起動時に `/dev/cu.*` デバイスを自動列挙
- F2 でポート選択画面を開き、リアルタイムで再スキャン（接続中でも切り替え可）
- 接続中にポートが抜けたら自動検出してエラー表示
- 自動再接続: 一定間隔でポートの復帰を試みる（オプション）
- DTR / RTS 信号を手動で ON/OFF できる（F3 設定画面内）

### 2. ポート設定

| パラメータ | デフォルト | 選択肢 |
|-----------|-----------|--------|
| ボーレート | 115200 | 300 / 1200 / 2400 / 4800 / 9600 / 19200 / 38400 / 57600 / 115200 / 230400 / 460800 / 921600 / カスタム |
| データビット | 8 | 5 / 6 / 7 / 8 |
| パリティ | None | None / Odd / Even / Mark / Space |
| ストップビット | 1 | 1 / 2 |
| フロー制御 | None | None / Hardware (RTS/CTS) / Software (XON/XOFF) |
| DTR 初期値 | OFF | ON / OFF |
| RTS 初期値 | OFF | ON / OFF |
| 受信タイムアウト | 100ms | 任意 ms |

### 3. 改行コード設定

送信と受信をそれぞれ独立して設定できる。

#### 送信改行コード

| 設定値 | 送信バイト | 用途例 |
|--------|-----------|--------|
| `CRLF` | `0x0D 0x0A` | Windows / AT コマンド全般 |
| `LF` | `0x0A` | Linux / Unix |
| `CR` | `0x0D` | 古い Mac / 一部組み込み機器 |
| `None` | （なし）| 改行なしで生バイト送信 |

- 入力バー左端に現在の設定を常時表示: `TX CRLF >`
- `Ctrl+E` でサイクル切り替え（CRLF → LF → CR → None → CRLF）

#### 受信改行コード（行分割の基準）

| 設定値 | 動作 |
|--------|------|
| `Auto` | `\r\n` / `\n` / `\r` を自動認識（推奨） |
| `CRLF` | `\r\n` のみで改行 |
| `LF` | `\n` のみで改行 |
| `CR` | `\r` のみで改行 |
| `None` | 改行なし（受信データを連続表示） |

### 4. 受信表示

- 受信データを UTF-8 でデコードして表示
- UTF-8 デコード失敗バイトは `·` (置換文字) または `[XX]` 形式で表示（切り替え可）
- タイムスタンプ表示オプション: `[HH:MM:SS.mmm]` を各行先頭に表示（μs 精度まで設定可）
- カラー表示:

  | 種別 | 色 |
  |------|---|
  | 受信行 | 白 |
  | 送信行 | シアン |
  | システム / 接続メッセージ | 黄 |
  | エラー | 赤 |
  | ハイライト一致行 | マゼンタ背景 |

- スクロール: 最大バッファ行数は設定可（デフォルト 10,000 行）
- 受信時に自動スクロール（スクロールロック中は無効、ステータスバーに `[LOCK]` 表示）

### 5. 表示モード切り替え（F4）

サイクル: `ASCII → HEX → SPLIT(ASCII+HEX) → ASCII`

| モード | 内容 |
|--------|------|
| ASCII | テキスト表示（デフォルト）|
| HEX | 生バイト列 + ASCII 対応表示 |
| SPLIT | 左ペイン ASCII / 右ペイン HEX の並列表示 |

### 6. 送信

- 入力バーで文字を編集してから Enter で送信
- 送信履歴: `↑↓` で過去の送信内容を呼び出し（最大 200 件）
- エスケープシーケンス展開: `\r` `\n` `\t` `\xHH` をバイト列に変換して送信
  - 例: `AT\r\n` → `41 54 0D 0A`
- ローカルエコー: 自分の送信内容をログ画面に表示（デフォルト ON、`Ctrl+R` でトグル）
- Raw モード: 入力を 1 文字ずつ即時送信（Enter 不要、`Ctrl+W` でトグル）
  - 組み込み機器の対話型シェル / REPL に便利

### 7. 画面クリア

| 操作 | 動作 |
|------|------|
| `Ctrl+L` | ログバッファ完全クリア（TUI の表示もリセット）|
| `Ctrl+K` | 表示だけクリア（バッファはメモリに保持、ログファイルは継続）|
| `Ctrl+X` | 接続を切断してバッファもクリア（ポート選択画面へ）|

> **Note:** クリア操作はログファイルに `[CLEARED]` マーカーを挿入するため、ファイルのデータは失われない。

### 8. 検索・フィルタ（F7）

- `/` キーでインクリメンタル検索バーを開く
- 受信バッファ内を正規表現で検索、ヒット行をハイライト
- `n` / `N` で次/前のヒットへジャンプ
- フィルタモード: ヒット行だけを表示（非マッチ行を折りたたむ）

### 9. マクロ（F6）

- TOML で定義した送信シーケンスを名前付きで登録
- マクロ選択画面からワンキーで実行
- ウェイト付きシーケンス: ステップ間に遅延を挿入できる

```toml
[[macros]]
name = "リセット"
steps = [
  { send = "\x1b[?25l", delay_ms = 0 },
  { send = "reset\r\n", delay_ms = 100 },
  { send = "show version\r\n", delay_ms = 500 },
]

[[macros]]
name = "AT テスト"
steps = [
  { send = "AT\r\n", delay_ms = 200 },
  { send = "AT+GMR\r\n", delay_ms = 200 },
]
```

### 10. 統計・診断パネル（`Ctrl+I` でトグル）

```
┌─ 統計 ─────────────────────────────────────────────────────┐
│ RX: 12,345 bytes  TX: 678 bytes  Error: 0  Uptime: 00:12:34 │
│ RX Rate: 1,024 B/s  Last RX: 0.3s ago  Buffer: 4,321 lines  │
│ DTR: ON  RTS: OFF  CTS: ON  DSR: ON  DCD: OFF  RI: OFF       │
└────────────────────────────────────────────────────────────-┘
```

- RS-232 制御線 (DTR/RTS/CTS/DSR/DCD/RI) の現在値をリアルタイム表示
- 受信/送信バイト数、エラーカウント、スループット

### 11. ログ保存

- F5 でログ記録の開始・停止をトグル
- ログファイルパスはコンフィグまたは起動引数で指定（デフォルト: `./easy_console_YYYYMMDD_HHMMSS.log`）
- ログフォーマットを選択:

  | フォーマット | 例 |
  |------------|---|
  | `text` | `[2026-04-05 12:34:56.123] RX: Hello, World!` |
  | `hex` | `[2026-04-05 12:34:56.123] RX: 48 65 6C 6C 6F` |
  | `raw` | 生バイト列をそのままファイルに書き出す（タイムスタンプなし）|

- クリア操作時に `--- [CLEARED at 12:34:56] ---` マーカーを挿入

### 12. クリップボードコピー

| 操作 | 動作 |
|------|------|
| `Ctrl+Y` | ログバッファ全体をクリップボードにコピー |
| `Ctrl+Shift+Y` | 選択行だけコピー（将来実装）|

### 13. プロファイル

- TOML 形式の設定ファイル (`~/.config/easy_console/profiles.toml`)
- 複数プロファイルを名前で管理
- 起動引数 `--profile <name>` でプロファイルを指定
- TUI 内からも切り替え可能（F3 設定画面）

```toml
[profiles.default]
port = "/dev/cu.usbserial-0001"
baud_rate = 115200
data_bits = 8
parity = "none"
stop_bits = 1
flow_control = "none"
tx_newline = "crlf"        # crlf / lf / cr / none
rx_newline = "auto"        # auto / crlf / lf / cr / none
timestamp = true
timestamp_precision = "ms" # ms / us
local_echo = true
scroll_buffer = 10000
log_format = "text"
view_mode = "ascii"        # ascii / hex / split
dtr_init = false
rts_init = false

[[profiles.default.macros]]
name = "AT テスト"
steps = [{ send = "AT\r\n", delay_ms = 200 }]
```

```

  -p, --port <PORT>         シリアルポートデバイスパス

      --log-format <FMT>    ログフォーマット (text/hex/raw) [デフォルト: text]
      --profile <NAME>      使用するプロファイル名
      --list-ports          利用可能なポートを表示して終了
  -h, --help                ヘルプを表示
  -V, --version             バージョンを表示
```

## 非同期アーキテクチャ

```
┌──────────────────┐  mpsc(RxEvent)  ┌────────────────────┐
│  serial          │ ──────────────> │                    │
│  reader task     │                 │   app loop         │ ──render──> ratatui
│  (spawn_blocking)│                 │   (main task)      │
│                  │                 │                    │
│  serial          │ <─────────────- │                    │
│  writer task     │  mpsc(TxCmd)    └────────────────────┘
│  (spawn_blocking)│                         ↑
└──────────────────┘                    crossterm
                                         keyboard
                                         events
```

- `tokio::sync::mpsc` で受信イベントをメインループに送る
- `tokio::sync::mpsc` で送信コマンドをライタータスクに渡す
- シリアル IO はスレッドブロッキングのため `tokio::task::spawn_blocking` を使用
- マクロ実行は独立した `tokio::task` として spawn し、ステップ間の `delay_ms` を `tokio::time::sleep` で処理

## エラーハンドリング

| 状況 | 挙動 |
|------|------|
| ポートが開けない | エラーポップアップ表示、ポート選択画面へ誘導 |
| ポートが抜けた | ステータスバーにエラー表示、自動再接続を試みる |
| UTF-8 デコード失敗 | `·` または `[XX]` 形式で表示（設定で切り替え）|
| ログファイル書き込み失敗 | ステータスバーに警告、記録を停止 |
| マクロ実行中に切断 | マクロを中断、エラーをログに表示 |
